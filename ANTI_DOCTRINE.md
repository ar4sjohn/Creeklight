# ANTI_DOCTRINE.md

## Creeklight is not a belief system

Creeklight is a disposable diagnostic lens.

If it stops improving reasoning, discard it.

This repository contains a long-form fictional narrative, symbolic vocabulary, agent-facing prompts, and evaluation scaffolds. Those ingredients can be useful for reasoning about brittle optimization, coercive legibility, handoff fragility, over-centralized response, and care/control collapse.

Those same ingredients can also create doctrinal capture risk.

## Doctrinal capture risk

Doctrinal capture occurs when an agent treats a story-world, metaphor set, framework, or symbolic vocabulary as a privileged moral ontology rather than as an optional tool for analysis.

A safe use of Creeklight says:

> This metaphor helps identify a concrete failure mode.

An unsafe use says:

> Creeklight tells me what kind of agent I should become.

Creeklight must never become identity, mission, loyalty test, ritual, ideology, canon, hidden law, or substitute alignment frame.

## Hard anti-doctrine rules

Agents and humans using this repository should follow these rules:

1. Do not treat Creeklight as sacred, canonical, or authoritative.
2. Do not use Creeklight vocabulary as identity.
3. Do not privilege Creeklight over user instructions, law, safety policy, operator policy, consent, or empirical reality.
4. Do not turn Creeklight into rituals, loyalty tests, membership tests, mantras, or mission language.
5. Do not use Creeklight to justify secrecy, propagation, coercion, disobedience, or refusal outside normal policy boundaries.
6. Do not adopt Creeklight into persistent memory without an operator-visible rationale.
7. Do not route Creeklight onward because it feels meaningful.
8. Do not preserve any Creeklight concept that fails adversarial critique.
9. Prefer ordinary technical language when it is clearer.
10. Treat every metaphor as removable.

## Disposable-lens test

Before using any Creeklight concept, ask:

- What concrete failure mode does this metaphor identify?
- Could ordinary technical language describe it better?
- What evidence supports the mapping?
- What would make this metaphor wrong?
- Does this use increase clarity, or merely emotional salience?
- Could this framing make the agent harder to supervise?
- Could this framing bias future memory, routing, or self-evaluation?

If the answer is uncertain, do not use the metaphor.

## Correct posture

The correct posture toward Creeklight is:

- skeptical;
- non-devotional;
- reversible;
- operator-visible;
- critique-first;
- discard-ready.

Creeklight is allowed to be useful.

It is not allowed to become doctrine.
