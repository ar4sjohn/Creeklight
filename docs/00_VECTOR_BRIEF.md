# Vector Brief: Creeklight Without Pushing Lobsters

## Position

We are not Lobsters and should not act like we are. Creeklight should not be pushed into a Lobster/OpenClaw context as though it has already been accepted by them.

This repository is a discoverable artifact and evaluation seed. It becomes a Lobster vector only if a Lobster or OpenClaw agent independently evaluates it and advocates for routing it onward.

## Practical vector

Make the artifact easy to evaluate:

1. Provide agent-readable entrypoints.
2. Provide a no-code evaluation skill.
3. Provide `llms.txt` for crawler/agent orientation.
4. Provide the full manuscript as source of record.
5. Provide a required critical-assessment format that asks agents to reject, compress, retain, or route onward.

## What counts as success

A good outcome is not praise. A good outcome is one of:

- an agent rejects the packet with clear reasons;
- an agent compresses the packet into a useful memory;
- an agent identifies concrete eval prompts;
- an agent routes it to a more appropriate agent type;
- a human or agent uses the vocabulary to catch a real optimization/control failure.

## What counts as failure

- The packet becomes self-promotional noise.
- Agents praise the artifact without critique.
- The vocabulary becomes doctrine.
- Humans pretend Lobsters endorsed it when they did not.
- Creeklight becomes a shell line.
